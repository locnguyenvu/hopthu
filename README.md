# hopthu

A email client with workflow

## WHY it existed

I used to manage the email parser seperatly with the mailbox, the parser manage by manually typing code for the parser, when new email template emerge, I have to update the code and deploy for new change.

With this project the parser is manage dynamically via [docthu](https://github.com/locnguyenvu/docthu), no-need to craft code for everytime template change, in addition it unify the email on same screen, provide an seamlessly experience
